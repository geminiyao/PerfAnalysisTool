【必须立即修复的 5 个 P0 问题】

问题 1️⃣ : SSE emit 时序竞态
  ├─ 文件: cli-executor.ts + analysis.ts
  ├─ 行号: L86-95, L119 (cli-executor) + L12-19, L88-96 (analysis)
  ├─ 症状: 前端收不到初期 preprocessing 日志
  └─ 根本原因: CLI 在 SSE 连接前启动，emit 被丢弃
  
  快速修复:
    在 executeCli() 前添加握手检查
    或缓存初期 emit 事件

问题 2️⃣ : stdout 缓冲区背压未处理
  ├─ 文件: cli-executor.ts
  ├─ 行号: L110-115
  ├─ 症状: 大型输出被截断，83s 运行但数据丢失
  └─ 根本原因: highWaterMark = 16KB，超过后背压
  
  快速修复:
    const child = spawn(cliCommand, args, {
      ...
      maxBuffer: 10 * 1024 * 1024,  ← 添加这行
    });

问题 3️⃣ : 缺少 'close' 事件监听
  ├─ 文件: cli-executor.ts
  ├─ 行号: L147-165
  ├─ 症状: 最后的数据可能丢失
  └─ 根本原因: 监听 exit 而非 close，stdout 还未完全关闭
  
  快速修复:
    改 child.on('exit', ...) 为 child.on('close', ...)

问题 4️⃣ : shell: true Windows 坑
  ├─ 文件: cli-executor.ts
  ├─ 行号: L113
  ├─ 症状: Windows 用户参数解析错误
  └─ 根本原因: cmd.exe 特殊字符处理
  
  快速修复:
    删除 shell: true 
    或改为: shell: process.platform === 'win32' ? 'powershell.exe' : true

问题 5️⃣ : SSE write() 无错误处理
  ├─ 文件: analysis.ts
  ├─ 行号: L92-93
  ├─ 症状: 大数据量时连接崩溃
  └─ 根本原因: write() 无 try-catch，背压无处理
  
  快速修复:
    const send = (event: ProgressEvent) => {
      try {
        if (!reply.raw.writable) return;
        const canContinue = reply.raw.write(
          `data: ${JSON.stringify(event)}\n\n`
        );
        if (!canContinue) {
          // 背压处理
          child?.pause?.();
        }
      } catch (err) {
        console.error(`SSE error: ${err.message}`);
      }
    };
