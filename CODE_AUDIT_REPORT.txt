问题 1: SSE emit 时序竞态导致数据丢失 (L86-95, L119)
─────────────────────────────────────────────────────
严重度: ★★★★★

时序问题：
  T0.1s:  spawn CLI → 立即 emit('preprocessing')
          sseClients 还是空 → emit 被丢弃 ❌
  
  T1.0s:  前端建立 SSE 连接
          sseClients[sessionId] 注册成功
          但初期数据已经丢失了 ❌

影响：前端看不到分析初期的任何日志

修复：缓存初期 emit，或延迟启动 CLI


问题 2: stdout 缓冲区背压未处理 (L110-115)
─────────────────────────────────────────────
严重度: ★★★★★

问题：
  - 默认 highWaterMark = 16KB
  - 当输出 > 16KB 时触发背压
  - stdout 暂停，数据堆积
  - 最终数据丢失或被截断

症状：CLI 继续运行 83s，但 Node.js 只读部分数据

修复：
  const child = spawn(cliCommand, args, {
    ...
    maxBuffer: 10 * 1024 * 1024,  ← 添加这行
  });


问题 3: 缺少 'close' 事件监听 (L147-165)
────────────────────────────────────
严重度: ★★★★★

问题：
  - 现在监听 exit 事件
  - exit 时 stdout 还没完全关闭
  - 最后的 JSON 数据可能丢失

修复：改用 'close' 事件
  child.on('close', (code, signal) => {
    // 所有 stdio 都已关闭
    if (!state.resolved) { doResolve(...); }
  });


问题 4: shell: true 在 Windows 上的参数解析错误 (L113)
──────────────────────────────────────────────
严重度: ★★★★★

问题：
  - Windows cmd.exe 的特殊字符处理
  - 中文路径可能被错误解析
  - 双引号和空格转义问题

修复方案 A：移除 shell: true
  spawn(cliCommand, args, { ... })  ← 删除 shell: true

修复方案 B：使用 PowerShell
  shell: process.platform === 'win32' ? 'powershell.exe' : true


问题 5: SSE write() 无错误处理 (analysis.ts L92-93)
──────────────────────────────────────────────
严重度: ★★★★★

问题：
  const send = (event: ProgressEvent) => {
    reply.raw.write(`data: ${JSON.stringify(event)}\n\n`);  ← 无保护
  };

缺陷：
  - 没有 try-catch
  - write() 返回 false 表示背压，无处理
  - 连接关闭时会崩溃

修复：
  const send = (event: ProgressEvent) => {
    try {
      if (!reply.raw.writable) return;
      const canContinue = reply.raw.write(
        `data: ${JSON.stringify(event)}\n\n`
      );
      if (!canContinue) {
        // 背压处理
        child?.pause?.();
      }
    } catch (err) {
      console.error(`SSE error: ${err.message}`);
    }
  };
